## Project Overview

Masclet is a multilingual chatbot about the Fallas festival in Valencia. It uses a 5-layer NLP pipeline (temporal в†’ CASCADE family classification в†’ direct regex matching в†’ Fuse.js fuzzy search в†’ guided fallback) over a centralized knowledge base (`src/data/knowledge.json`) with support for ES (256 intents), VA, EN, and FR.

**Stack**: Vanilla JS (no transpilation/bundling), Gulp 5, SCSS, Fuse.js, Compromise.js, Lodash (all loaded via CDN in browser).

## Commands

```bash
npm install          # Install dependencies
npm run dev          # Build + BrowserSync on localhost:3000 + watch
npm run build        # Clean build to dist/
npm run sass         # Compile SCSS only
npm test             # Run Jest suite (197 tests)
```

Test command requires the `--experimental-vm-modules` flag (already configured in package.json).

## Architecture

### Source vs Build

All source lives in `src/`. The `dist/` directory is generated by Gulp вЂ” never edit it directly. JS is copied as-is (no transpilation). SCSS compiles to expanded + minified CSS. Images get `.webp` and `.avif` variants via Sharp.

### Two HTML Surfaces

- `src/index.html` вЂ” Full demo page with draggable/minimizable chat window
- `src/chatbot.html` вЂ” Standalone version for iframe embedding via `masclet-embed.js`

### Core Runtime (`src/js/app.js`)

Single file (~2000 lines) containing all chatbot logic. Key systems:

- **LANGUAGE_CONFIGS** (lines ~186вЂ“343): Per-language UI strings, temporal patterns, greetings, metadata
- **CASCADE patterns** (lines ~604вЂ“667): Regex patterns for 8 semantic families Г— 4 languages (`conversation`, `personality`, `events`, `history`, `organization`, `logistics`, `gastronomy`, `attire`)
- **Fuse integration** (lines ~668вЂ“1493): Weighted multi-field search with configurable thresholds (`fuseDirect=0.60`, `fuseGuidance=0.78`)
- **Response orchestrator** вЂ” `resolveResponseStrategy()`: Runs the 5-layer pipeline

### Knowledge Base (`src/data/knowledge.json`)

Language-keyed object with nested categories. The runtime flattens it recursively via `flattenKnowledgeBase()`, tagging each entry with `kbPath` for CASCADE family inference.

**Entry fields**: `trigger` (string, array, or `regex:` prefixed), `answer`/`text`, `keywords`, `followUps`, `image`/`imagen`, `images`, `video`.

### Trigger Formats

- Plain text: `"hola masclet"` вЂ” auto-converted to anchored regex
- Explicit regex: `"regex:^\\s*hola\\s*$"` вЂ” compiled as-is
- Array: `["hola", "hey"]` вЂ” each item becomes a separate regex

## Key Conventions

- **Language**: Code uses Spanish names for functions/variables (`respuestas`, `buscarConFuse`, `respuestasPlana`)
- **CSS**: BEM methodology, SCSS with SMACSS-inspired structure (abstracts/base/components)
- **No transpilation**: JS must be valid ES2020+ for modern browsers
- **All functions are global** in browser context (no module system)

## Working with Knowledge

When adding/modifying intents:

1. Edit `src/data/knowledge.json` under the appropriate language and category
2. Use `regex:` prefix for new regex triggers (legacy bare-regex triggers still work)
3. `\w` doesn't match accented chars in JS вЂ” use `[a-zГЎГ©Г­ГіГєГјГ±Г§]` for Spanish/Valencian
4. Anchor patterns with `\s*[?Вї!ВЎ.]*\s*$` to tolerate punctuation
5. Add corresponding tests in `tests/nlp.test.js`
6. Run `npm test` to validate

Adding a new language requires changes in: `knowledge.json`, `LANGUAGE_CONFIGS` in `app.js`, language selector in both HTML files, and all CASCADE/temporal pattern objects.

## Testing

Tests in `tests/nlp.test.js` load `knowledge.json` directly, flatten the KB, and test each NLP layer independently. Test blocks: helpers/utilities (~20), temporal resolution (~4), CASCADE (~8), direct regex matching (~140), Fuse fuzzy + fallback (~25).

## Runtime Debug Parameters (URL or localStorage)

- `?lang=es|va|en|fr` вЂ” force language
- `?debugFuse=1` вЂ” enable Fuse logging
- `?fuseDirect=0.60` / `?fuseGuidance=0.78` вЂ” override score thresholds
- `?kbUrl=/path/to/kb.json` вЂ” override knowledge base URL
